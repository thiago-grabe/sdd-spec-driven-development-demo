# Story 1.2: Summarize endpoint end-to-end

## Status

Draft

## Story

**As a** client application,
**I want** to POST text and receive a concise summary as JSON, with predictable errors,
**so that** I can show summaries to my users without embedding any model logic or credentials.

## Acceptance Criteria

1. Non-empty `text` → `200` with `summary`, `word_count`, `model`, `request_id`; `word_count`
   matches the summary. (FR1, FR2, FR7)
2. Empty/whitespace `text` → `422`, no model call. (FR3)
3. `text` > 10,000 chars → `413`, no model call. (FR4)
4. Model error/timeout → `502` with `{error, request_id}`, no leaked internals. (FR5)
5. Every response carries `request_id` and the `X-Request-ID` header. (FR6)
6. `max_words` is passed to the model as a soft target; a successful request makes exactly one model
   call. (FR8, FR9)
7. A seeded marker in `text` appears in no log record and no response body. (NFR1)

## Tasks / Subtasks

- [ ] Implement Pydantic models in `models/schemas.py` (AC: 1)
  - [ ] `SummarizeRequest` (text min length 1; max_words 1–1000 default 150)
  - [ ] `SummarizeResponse`, `ErrorResponse`
- [ ] Implement `services/summarizer.py` (AC: 1, 6)
  - [ ] Exactly one `chat.completions.create` call, 30s timeout, ≤1 bounded retry
  - [ ] Fold `max_words` into the system prompt; raise `SummarizerError` on failure
  - [ ] Inject the client via a FastAPI dependency so tests can mock it
- [ ] Implement request-id middleware in `main.py` (AC: 5)
- [ ] Implement `api/routes.py` (AC: 1, 2, 3, 6)
  - [ ] Validate → 10k 413 guard before any call → call → compute `word_count` → build response
- [ ] Implement `api/errors.py` (AC: 2, 3, 4)
  - [ ] Map validation→422, `PayloadTooLargeError`→413, `SummarizerError`→502 → `ErrorResponse`
- [ ] Tests, model client mocked (AC: 1–7)
  - [ ] Happy path, 422, 413, 502, X-Request-ID, exactly-one-call, privacy marker

## Dev Notes

Use the data models, status-code mapping, and source tree from `docs/architecture.md`. The 10k cap
is a **route-level 413**, not a Pydantic `max_length` (which would surface as 422 — wrong for FR4).
The privacy test (AC 7) is the load-bearing guardrail; build the logger and error handler so the
submitted text cannot reach them.

### Testing

- Locations: `tests/unit/test_schemas.py`, `tests/unit/test_summarizer.py`,
  `tests/integration/test_summarize_route.py`. Framework: `pytest` + httpx TestClient.
- The OpenAI client is mocked in `conftest.py`; no live calls (NFR5).
- Assert behavior and contract, not exact summary wording (model is non-deterministic).
