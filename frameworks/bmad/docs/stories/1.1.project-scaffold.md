# Story 1.1: Project scaffold and configuration

## Status

Approved

## Story

**As a** developer,
**I want** a typed, tested, lint-clean service skeleton with environment-driven configuration,
**so that** every later story has a consistent place to add behavior and CI can run offline.

## Acceptance Criteria

1. Dependencies install via `uv`; `ruff check` passes; `pytest` collects zero tests without error.
2. Settings (key, model, `timeout=30`, `cap=10000`, `retries=1`) load from environment variables
   with documented defaults; no secret is logged.
3. The logger emits metadata only (request id, length, status, latency, model) and is structurally
   unable to emit the submitted text.

## Tasks / Subtasks

- [ ] Create the package structure (AC: 1)
  - [ ] `src/summarizer_service/` with `core/`, `models/`, `services/`, `api/` and `__init__.py`s
  - [ ] `pyproject.toml` with FastAPI, Pydantic v2, openai, uvicorn, pytest, ruff
- [ ] Implement `core/config.py` (AC: 2)
  - [ ] `Settings` reads `OPENAI_API_KEY`, model, timeout, cap, retries from env with defaults
- [ ] Implement `core/logging.py` (AC: 3)
  - [ ] Metadata-only formatter; add a unit test that a seeded marker cannot be emitted
- [ ] Verify tooling (AC: 1)
  - [ ] `uv sync`, `ruff check`, `ruff format --check`, `pytest` all succeed
